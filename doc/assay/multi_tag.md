# Tag Workflow (CLindex™ Single-cell Multiplexing)

The tag workflow is specifically designed to analyze single-cell multiplexing data generated with kits such as CLindex™.

---

## Overview and Prerequisites

Before initiating the tag workflow, you must first complete the corresponding single-cell transcriptome (scRNA-Seq) analysis using CeleScope.

The results from the RNA pipeline serve as the foundation for cell identification in the tag pipeline.

---

## Mapfile Configuration

For the tag assay, the mapfile is a tab-delimited text file that requires a minimum of 4 columns:

| Column | Name              | Description |
|--------|------------------|-------------|
| 1st    | Files Prefix     | The prefix of the Fastq files (R1 and R2). |
| 2nd    | Fastq Path       | The directory path where the Fastq files are located. |
| 3rd    | Sample Name      | The name of the sample, used as the prefix for all output files. |
| 4th    | Matched Directory | Required. The output directory path from the previously completed CeleScope scRNA-Seq run. |

---

## Running the Pipeline

The recommended method for execution is using the `multi_tag` command to generate task scripts:

```bash
multi_tag \
    --mapfile ./tag.mapfile \
    --barcode_fasta ./CLindex_TAG.fasta \
    --fq_pattern L25C15 \
    --mod shell
```

Key Parameters

--barcode_fasta
Path to a FASTA file containing the expected Tag sequences.  It will check the mismatches between tag barcode sequence in R2 reads with all tag barcode sequence in barcode_fasta. It will assign read to the tag with mismatch < threshold. If no such tag exists, the read is classified as invalid.

--fq_pattern
The structural pattern of the Read 2 sequence.
For CLindex™ kits, the default is L25C15 (Linker 25bp, Barcode 15bp).

--split_matrix
If this flag is used, the pipeline will generate separate gene expression matrices for each assigned tag.

## Split Matrix Output Format

The matrices generated by the split_tag step (located in the 06.split_tag directory) follow the standard 10x Genomics-compatible format.

Each output folder contains three files:

- matrix.mtx.gz — Sparse expression matrix
- barcodes.tsv.gz — List of identified cell barcodes
- features.tsv.gz — List of gene IDs and gene names

Compatibility: These outputs can be directly loaded into downstream analysis tools such as Seurat and
Scanpy.
